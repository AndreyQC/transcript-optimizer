## 🎯 Цель

Две связанные части:

**Часть A — просмотр.** На правой панели (очищенный результат), в строке с кнопками
справа от «Синхронизировать», добавить кнопку-переключатель **«Свернуть реплики»**.
По нажатию — трансформировать отображаемый очищенный транскрипт: если один спикер
говорит несколько фраз подряд, убрать избыточные временные метки, оставив лишь
**первую** (как открывающую `[time]>`) и **последнюю** (как закрывающую `<[time]`)
реплику блока.

**Часть B — саммари.** Свёрнутый транскрипт становится **равноправным третьим
источником** в режиме «Саммари» (наряду с `raw` и `cleaned`): появляется третья
кнопка «Саммари (collapsed)», результат живёт в store на тех же правах, доступен
для потока/рендера/сохранения и участвует в Diff.

### Пример

**До:**
```
Andrey Potapov (голос 1):
[00:30:59] Спасибо.
[00:31:00] Ты мне как бы Аларчик просто открывался.
[00:31:04] Нужно просто зайти в Cursor и посмотреть.
[00:31:07] Спасибо тебе большое.
[00:31:09] А я тебя потом прошу: а можно мне же шарить между чатики?
[00:31:15] Такие подсмотреть.
```

**После:**
```
Andrey Potapov (голос 1):
[00:30:59]>
Спасибо.
Ты мне как бы Аларчик просто открывался.
Нужно просто зайти в Cursor и посмотреть.
Спасибо тебе большое.
А я тебя потом прошу: а можно мне же шарить между чатики?
Такие подсмотреть.
<[00:31:15]
```

### Дополнительные требования к трансформации
- Удалить идущие подряд знаки препинания `,` (запятая), возникающие при склеивании
  фраз (например, если фраза кончалась на `,`, а следующая начиналась после метки).
- Убрать подряд идущие несколько пробелов (нормализовать в один).

### Критерий готовности
**Часть A (просмотр):**
1. В шапке правой панели (`.pane-header`), справа от «Синхронизировать», появилась
   кнопка-переключатель «Свернуть реплики».
2. При нажатии Monaco справа показывает свёрнутую форму; при повторном — исходный
   очищенный текст. Состояние (свёрнуто/развёрнуто) хранится в store.
3. Метка блока: первая реплика получает `[time]>` на отдельной строке перед текстом,
   последняя — `<[time]` на отдельной строке после текста; промежуточные метки
   убраны, тексты реплик склеены через перевод строки.
4. Блок из **одной** реплики сворачивается в: `[time]>` / текст / `<[time]` (метка
   дублируется с обеих сторон — допустимо; см. «Граничные случаи»).
5. После склейки удаляются повторы `,` и схлопываются множественные пробелы.
6. Сворачивание НЕ мутирует `cleanResult.cleanedText` — это проекция для отображения.

**Часть B (саммари):**
7. В режиме «Саммари» появилась третья кнопка «Саммари (collapsed)», доступная при
   наличии `cleanResult` (как «Саммари (cleaned)»).
8. `collapsed` — равноправный источник: результат накапливается в store
   (`summaryCollapsed`), переживает размонтирование `SummaryView`, доступен во
   вкладках «Поток»/«Результат» и сохранении `.md`.
9. Diff работает для любой пары источников (см. зафиксированное решение по Diff).

**Общее:**
10. `tsc --noEmit` и `vite build` — зелёные; ручной тест на образце подтверждает вид.

### ВНЕ MVP (бэклог, НЕ делать в этом проходе)
- Кнопка экспорта свёрнутой формы в файл (сейчас экспортируется только stats).
- Настройка «по умолчанию сворачивать» в settings.yaml.
- Сворачивание по порогу длительности (например, только блоки > N секунд).
- Кастомные маркеры `[time]>` / `<[time]` (выбор стиля пользователем).
- Отдельный режим Diff-матрицы (показ всех 3 пар одновременно).
- Сохранение свёрнутого состояния просмотра между перезапусками приложения.

## 📐 Дизайн функции `collapseTimemarks`

### Алгоритм (построчный FSM)

```
вход: cleanedText (плоский, с шапкой/блоками спикера)
выход: свёрнутый текст

состояния: header | between | inBlock
accumulators:
  out: string[]                 — выходные строки
  blockTimes: string[]          — таймштампы реплик текущего блока
  blockTexts: string[]          — тексты реплик текущего блока

1. нормализовать переводы строк (\r\n → \n), split по \n
2. для каждой строки:
   - пустая строка → если inBlock, flushBlock(); push "" в out
   - заголовок спикера (RE_SPEAKER, не utterance) → flushBlock(); push в out; перейти inBlock
   - реплика [time] text → добавить time/text в аккумуляторы блока
   - прочее → push в out как есть (шапка, мусор)
3. в конце — flushBlock() для последнего блока

flushBlock():
  если blockTexts пуст → ничего
  иначе:
    first = blockTimes[0], last = blockTimes[длина-1]
    out.push(`[${first}]>`)
    out.push(...blockTexts)         // тексты реплик, каждый на своей строке
    out.push(`<[${last}]`)
    очистить аккумуляторы
```

### Пост-обработка склейки (отдельный проход по выходной строке)

После сборки — две замены, **именно в таком порядке**:

1. **Удалить повторы запятой**: Regex `,{2,}` → `,`.
   - Покрывает случай, когда фраза кончалась на `,`, а соседняя тоже добавила `,`.
   - Внимание: требование ТЗ — «удалить идущие подряд знаки препинания , (запятая)».
     Применяем только к запятым (не ко всем знакам препинания — точки/восклицательные
     сохраняем, иначе можно потерять границы предложений).

2. **Схлопнуть множественные пробелы**: Regex `[ \t]{2,}` → ` ` (один пробел).
   - На случай остатков после удаления меток/склейки. Не трогает переводы строк.

Обе замены применяются к **финальной строке целиком** (не к отдельным репликам) —
дёшево и покрывает границы между репликами.

> **Почему не прямо в FSM-сборке:** нормализация на финальной строке проще и
> не размазывает логику по шагам. FSM собирает структуру, пост-обработка чистит
> артефакты склейки.

### Граничные случаи (явно)

| Случай | Поведение |
|---|---|
| Блок из **1 реплики** | `[time]>` / текст / `<[time]` (та же метка с обеих сторон). Допустимо по ТЗ; альтернатива «не сворачивать одиночные» отвергнута — единообразие важнее. |
| Блок из **2 реплик** | `[t1]>` / текст1 / текст2 / `<[t2]`. Корректно. |
| **Шапка** перед первым спикером | Пробрасывается как есть (каждая строка в `out`). |
| **Пустые строки** между блоками | Сохраняются (разделитель). Внутри блока в `cleanedText` их нет (сборка в `applyRules` не вставляет). |
| **Дублирующиеся таймштампы** в блоке | Не особый случай — берём первый и последний как есть. |
| Строка, не матatching ни speaker, ни utterance внутри файла | Пробрасывается как есть (defensive). |
| `cleanedText` пустой | Возвращается пустая строка. |

## ⚠️ Риски и нюансы

1. **Синхронизация по таймштампу в свёрнутом виде.** Helper'ы ищут `[…` в начале
   строки. Строка `[00:30:59]>` матчится — OK. Но строки текста реплик между
   маркерами НЕ имеют таймштампа → `timeAtLine` для них вернёт метку открывающего
   маркера (ближайший `[…]` вверх), что semantically верно (они все в одном блоке
   с этой начальной меткой). Если выяснится, что синхронизация теряется — зафиксировать
   как баг и отдельно починить (бэклог).
2. **Тернарники по `sourceTab` в SummaryView.** Сейчас логика «raw vs cleaned»
   построена на булевых тернарниках; добавление третьего источника `collapsed`
   требует переключения на `switch`/map. Если пропустить — `collapsed` молча
   упадёт в ветку cleaned (покажется cleaned-результат). Шаг 7 оговаривает это явно.
3. **`runSummary("collapsed")` и флаг `collapseEnabled`.** collapsed-саммари НЕ
   зависит от того, включено ли сворачивание в просмотре — функция вызывается
   напрямую. Это сознательное решение (см. зафиксированное): пользователь может
   сравнивать саммари очищенного и свёрнутого, не трогая панель транскрипта.
4. **Diff при 3 источниках.** Жёсткий raw↔cleaned DiffEditor заменяется
   переключателем пары. Следить, чтобы `canDiff` и автопереключение
   (`diffOfferedRef`) корректно работали с «≥2 готовых результатов», а не «ровно 2».
5. **Производительность.** `collapseTimemarks` — O(n) по строкам, `useMemo` по
   `cleanResult` в просмотре; в саммари — однократно при `runSummary`. Беспокоиться
   не о чем.
6. **Пустые блоки / битый ввод.** FSM defensive: пустой блок не флашится, мусорные
   строки пробрасываются. Тест на минимальном/битом файле желателен.
7. **Регистр/кодировка маркеров `<` `>`.** Это ASCII, проблем с Unicode нет. Но
   убедиться, что Monaco `plaintext`-режим не пытается подсветить `<…>` как HTML —
   `language="plaintext"` гарантирует отсутствие подсветки тегов.
