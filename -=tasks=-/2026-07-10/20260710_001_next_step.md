## 🏗️ Архитектура (кратко, для ориентации)

```
app/src/
├─ App.tsx                    режим: «Словари» | «Транскрипт» (переключатель вверху)
├─ types/dictionaries.ts      TS-типы 6 схем + FILE_CONTRACT
├─ store/
│  ├─ dictionaries.ts         Zustand: dir/entries/activeKind/undoState + getGlossaryCategories
│  └─ transcript.ts           Zustand: transcript/cleanResult/cleanDirty
├─ engine/                    ← ЧИСТЫЕ функции, тестируются без UI
│  ├─ parser.ts               строчный КА: header/speaker/utterance, якорь=lineNo (time НЕ ключ)
│  ├─ tokenizer.ts            mixed ru/en, дефисы, апострофы; tokenize() + norm()
│  ├─ rules.ts                effectiveWhitelist + applyRules; colShift=11 для decorations
│  └─ types.ts                ParsedTranscript/CleanResult/Decoration/ReplacementHit/CleanStats
├─ lib/
│  ├─ fs.ts                   Tauri fs/dialog: pickDir/readFile/writeFile/joinPath/detectDictionaries
│  ├─ validate.ts             ручные валидаторы → Monaco markers (позиции через LineCounter)
│  ├─ yaml-edit.ts            AST-правка через CST (addEntry/deleteEntry, сохраняет комментарии)
│  ├─ glossary-export.ts      replacements+glossary → GLOSSARY.md
│  └─ stats-export.ts         CleanResult → Markdown
└─ components/
   ├─ Toolbar.tsx             mode-aware: словари (open/save/export) | транскрипт (open/clean/sync)
   ├─ DictionaryTabs.tsx      вкладки по TAB_ORDER
   ├─ YamlEditor.tsx          Monaco YAML + markers; getGlossaryCategories через useMemo
   ├─ EditPanel.tsx           формы add/delete + preview-diff + undo
   └─ TranscriptView.tsx      2 панели Monaco (orig с decorations / clean) + StatsPanel + sync
```

### Ключевые паттерны (НЕ НАРУШАТЬ)
- **Zustand-селекторы НИКОГДА не возвращают новый объект/массив/Set** (урок из LESSONS_LEARNED §3 — бесконечный ре-рендер). Производные коллекции — через `useMemo` от стабильной подписки. `getGlossaryCategories(entries)` — пример: вызывается в `useMemo`, НЕ как селектор.
- **Движок — чистые функции** в `engine/`, тестируются через `npx tsx` smoke-скрипты без UI.
- **On-demand философия**: очистка по кнопке, синхронизация панелей по кнопке (пользователь сам выбрал этот подход — не авто).
- **Decorations**: колонки включают сдвиг `colShift = "[time] ".length` (=11), т.к. Monaco показывает полную строку с префиксом.

## 🎯 СЛЕДУЮЩИЙ ШАГ: Этап 3 — контекстное меню ПКМ → словарь

**Это приоритет №1.** Пользователь видит красное OOV → ПКМ → «добавить в whitelist» → слово перестаёт краснеть после пере-очистки. Максимальная ценность для сценария «дополняю словарь».

### Требования (из idea.md §4.3, §8.3.2)
Действия по правому клику на выделении (слово ИЛИ фраза — idea §8.3.2):
- **«Добавить в whitelist»** → `detector_whitelist.yaml` (слово корректное)
- **«Удалить всегда (filler)»** → `filler.yaml → filler_words` (или filler_phrases если фраза)
- **«Заменить на…»** → `replacements.yaml` (мини-форма: ввод `to`, выбор `label` из glossary dropdown)
- **«Указать лемму»** → `lemma_irregular.yaml` (слово → лемма)
- **«Не трогать (keep)»** → `filler.yaml → keep_override`

### Архитектурные решения (УЖЕ ПРИНЯТЫ, не пересомневаться)
- AST-правка через `yaml-edit.ts` (`addEntry`) — уже реализована в EditPanel, переиспользовать.
- `label` обязателен для replacements → в мини-форме dropdown категорий (как в EditPanel).
- Перед записью — preview-diff (паттерн из EditPanel), после — пере-парсинг + авто-валидация.
- Действие доступно в режиме «Транскрипт», по выделению в **оригинальной** панели (где decorations).

### Что нужно сделать (задачи 3.x)
- **3.1** Зарегистрировать context-menu в Monaco оригинальной панели (`editor.addAction` или `onContextMenu`). Получить выделение (selection → строка + колонки).
- **3.2** Маппинг «выделение → тип действия»: слово → все 5 действий; фраза (несколько слов) → filler_phrase/replacements/keep.
- **3.3** Мини-формы действий (reuse EditPanel-логику или вынести в общий модуль). Для replacements — ввод `to` + `label` dropdown.
- **3.4** Применить через `addEntry` (yaml-edit) → `applyEdit` (dictionaries store) → dirty-флаг → пользователь видит правку, сохраняет, пере-очищает.
- **3.5** tsc + vite build + smoke.

### Критерий готовности
ПКМ по красному OOV-слову → «добавить в whitelist» → сохранить → «Очистить» → слово больше не красное.
