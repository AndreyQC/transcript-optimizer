## 🏗️ Что это за проект

Desktop-приложение для интерактивной оптимизации транскриптов (расшифровок речи) и управления YAML-словарями очистки. Стек: **Tauri 2 + React 19 + TypeScript + Vite + Monaco Editor + Zustand + `yaml` (eemeli, CST)**.

Четыре top-level режима в UI:
1. **Словари** — редактор 6 YAML-файлов с автодетектом, валидацией, AST-правкой, undo, экспортом глоссария.
2. **Транскрипт** — двухпанельный просмотр: оригинал слева, очищенный справа; движок правил (filler/replacements/whitelist); статистика OOV; свёртка временных меток; ПКМ → добавление в словарь.
3. **Саммари** — LLM-summary через OpenAI-совместимый API (SSE-стриминг из Rust); raw/cleaned/collapsed источники; Markdown/Mermaid preview; Diff.
4. **Markdown** — *новый* split-view редактор произвольных `.md` (Monaco + live preview + Ctrl+S + сохранить как).

## 🔒 Ключевые архитектурные решения (не нарушать)

1. **Zustand-селекторы возвращают только плоские поля** — никогда не возвращают новый объект/массив/Set из селектора. Производные коллекции — через `useMemo` от стабильной подписки. См. `LESSONS_LEARNED.md §3`.
2. **Движок — чистые функции** в `engine/`. Тестируются через временные `.mjs` с Node ESM loader-хуком (без tsx).
3. **YAML-правка через AST/CST** (`yaml` пакет). Никакой строковой правки — сохраняем комментарии и стиль.
4. **LLM-стриминг через Rust-команду** (`stream_chat`), не через frontend-fetch — баг плагина HTTP с буферизацией.
5. **API-ключ из OS env `OPENAI_API_KEY`** (Rust `std::env::var`), не localStorage/keychain.
6. **On-demand философия** — сохранение, очистка, синхронизация по кнопке, не авто.
7. **CSP `null`** — обязательно для Monaco + Mermaid.
8. **Состояние, привязанное к сессии, живёт в store** — не в локальном `useState`, иначе размонтирование компонента теряет данные (Markdown-фича и саммари — пример).
