# Transcript Optimizer

Desktop-приложение для интерактивной оптимизации транскриптов (расшифровок речи) и сопровождения YAML-словарей очистки. Совмещает три роли:

1. **Рабочее место корректора** — оригинальный транскрипт и очищенный результат с подсветкой подозрительных слов.
2. **Редактор словарей** — наполнение и валидация YAML-справочников; правки «в один клик» с сохранением структуры и комментариев файла.
3. **Глоссарий проекта** — словарь замен превращается в человекочитаемый глоссарий (с категориями и определениями) и экспортируется в Markdown.

> **Статус:** в разработке. Этапы 0-4 реализованы: [редактор словарей](#статус-реализации), движок очистки + просмотр транскрипта со статистикой OOV, контекстное меню ПКМ → словарь, LLM-summary.

---

## Возможности

- **Автодетект словарей** по контракту имён: открыл папку — приложение нашло все файлы и распознало их схемы.
- **Monaco YAML-редактор** с подсветкой ошибок валидации прямо в тексте (красные/жёлтые волны на конкретных полях).
- **Валидация схем и ссылочной целостности**: обязательный `label`, принадлежность категориям глоссария, уникальность канонических терминов, конфликты вариантов, цепочки замен.
- **AST-правка через формы** — добавление/удаление записей с сохранением комментариев, порядка и стиля исходного YAML (через CST).
- **Undo** для структурных правок; подтверждение перед закрытием с несохранёнными изменениями.
- **Экспорт глоссария** в `GLOSSARY.md`, сгруппированного по категориям.
- **Движок очистки транскрипта**: применение правил (replacements, filler, лемматизация, OOV-детекция) с подсветкой подозрительных слов прямо в оригинальном тексте.
- **Просмотр транскрипта**: две панели (оригинал ↔ очищенный результат) с синхронизацией по таймштампу; левая панель — со вкладками «Оригинал» и «Статистика».
- **Свёртка временных меток**: кнопка «Свернуть реплики» убирает избыточные таймштампы в блоках одного спикера, оставляя открывающую `[time]>` и закрывающую `<[time]` метки.
- **Статистика OOV**: грид слов, не найденных ни в одном словаре и в replacement, с частотой и множественным селектом — добавление выбранных слов в whitelist одним кликом (батч), либо в replacements через диалог (по одному).
- **Контекстное меню ПКМ** по выделению в транскрипте → добавление слова/фразы в любой из словарей с превью diff перед применением.
- **LLM-summary**: режим саммари по трём источникам (raw / cleaned / collapsed) через OpenAI-совместимый API (стриминг, Markdown + Mermaid, diff между любой парой результатов).
- **Тёмная и светлая тема**.

## Формат словарей

Приложение работает с каталогом из 6 YAML-файлов. Имена фиксированы (контракт автодетекта):

| Файл | Назначение |
|---|---|
| `settings.yaml` | Глобальные настройки пайплайна |
| `glossary.yaml` | Категории для `label` (enum: id + title) |
| `filler.yaml` | Слова/фразы-заполнители и междометия для удаления |
| `replacements.yaml` | Замены (`from → to`) + лемматизационные замены — **он же глоссарий проекта** |
| `lemma_irregular.yaml` | Карта «слово во флексии → лемма» |
| `detector_whitelist.yaml` | Белый список часто встречающихся слов |

`replacements.yaml` — двойного назначения: механизм замен и глоссарий. `label` обязателен (id категории из `glossary.yaml`), `description` опционален (превращает запись в термин глоссария). Подробности и примеры схем — в [`-=tasks=-/2026-07-09/20260709_001_idea.md`](./-=tasks=-/2026-07-09/20260709_001_idea.md) (раздел 10).

## Запуск (разработка)

### Требования

- [Node.js](https://nodejs.org/) ≥ 20
- [pnpm](https://pnpm.io/)
- [Rust](https://www.rust-lang.org/tools/install) (stable)
- Системные зависимости Tauri: на Windows — WebView2 (предустановлен в 10/11) и MSVC build tools.

### Установка и запуск

```bash
cd app
pnpm install
pnpm tauri dev
```

Откроется окно приложения. Нажми **«Открыть папку словарей»** и укажи каталог с YAML-файлами (например, `sample/transcript_optimizer`).

### Проверка сборки (без GUI)

```bash
cd app
pnpm exec tsc --noEmit   # типы
pnpm exec vite build     # фронтенд
cargo check --manifest-path app/src-tauri/Cargo.toml  # бэкенд
```
