## Key decisions & constraints

- **Дыра §6 закрыта в движке**, а не обойдена патчем агрегатора. Многословные `from` теперь матчатся подстрокой через `wordBoundaryRe` (как `filler_phrases`), а не пытаются попасть в `Map<слово>` через ключ с пробелом.
- **Единый `.replace(re_gi, fn)`** для фраз — JS не ре-сканирует вставленный replacement → нет двойной подстановки при `to ⊃ from` (`English Club` → `English Club community`). Проверено явно.
- **Фразы сортируются по убыванию длины** + `covered` spans — корректно для перекрытий.
- **Числовой фильтр в агрегаторе, не в движке** — по принципу §6: движок = источник истины для decorations (числа подсвечены в transcript), фильтрация шума — в `buildOovRows`.
- **§6-патч `buildOovRows` сохранён** — безвредный no-op для replacement-фраз, но нужен для `filler_phrases`.
- **Новые категории — только данные**: `label` как строка-ссылка на id из `glossary.yaml`; UI/валидация/экспорт data-driven.
