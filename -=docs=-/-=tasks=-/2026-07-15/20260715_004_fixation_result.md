## Lessons Learned (дополнение)

### 15. Ручные правки sample-данных — отдельный коммит

Когда в процессе проверки UI-фичи пользователь руками правит sample-словари, это
**отдельный коммит** с префиксом `test(sample): ...`:

- Держит `git log --grep=^feat` чистым: видно только историю кода приложения.
- Не путает ревью фичи с ревью данных.
- Легко откатить данные без отката кода: `git checkout HEAD~1 -- sample/`.

Привычка «всё в один коммит» работает для небольших правок, но в проекте с частыми
данными-фидбэками от пользователя она засоряет историю.
